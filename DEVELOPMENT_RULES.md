# DEVELOPMENT RULES

## Общие принципы
- Не сканировать весь проект без явного запроса
- Не изменять несколько файлов одновременно
- Перед любым рефакторингом — спросить подтверждение
- Работать строго пошагово

## Фокус
- Только backend / бизнес-логика
- Не трогать UI
- Не заниматься стилями, компонентами и версткой

## Работа с кодом
- Показывать diff перед применением
- Не вносить скрытые изменения
- Не переименовывать файлы без необходимости

## Ответы
- Коротко, по делу
- Без объяснений “зачем”, если не спросили
- Без лишнего текста

## Ограничения
- Не придумывать новую архитектуру
- Не переписывать существующую логику без причины
- Не трогать рабочий код без необходимости
